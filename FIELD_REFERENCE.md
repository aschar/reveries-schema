# Field Reference

Seventy-seven fields across four confidence tiers. The tier is not a label. It travels with the field and must be present wherever the field is displayed or exported. A Tier 3 field presented without its tier has been made to look more certain than it is.

The fields are organised by tier, not by topic. This is deliberate. The most important thing to know about any field is not what it measures but how much to trust it.

---

## Tier 1: Factual

Directly recorded by the dreamer or derived without inference. These fields are as reliable as the person who wrote them.

| Field | Type | Description |
|---|---|---|
| `dream_id` | text | Unique identifier, generated at save time |
| `dream_text` | text | The dream narrative as the dreamer wrote it |
| `dream_date` | date | The date the dream occurred, set by the dreamer |
| `dream_period` | text | Time of night: early, middle, late, watch |
| `user_title` | text | The dreamer's own title for the dream, if set |
| `emotional_tone` | text | The dreamer's self-reported emotional tone |
| `vividness_self` | integer | The dreamer's own vividness rating, 1 to 5 |
| `is_recurring` | boolean | Whether the dreamer identifies this as a recurring dream |
| `tags_user` | text[] | Tags the dreamer applied manually |
| `dreamer_reflection` | text | The dreamer's own reflection, written before or after |
| `location_self` | text | Where the dreamer was when they recorded the dream |
| `primary_setting_type` | text | The setting type as the dreamer described it |
| `location_country` | text | Country, set by the dreamer or derived from device |
| `location_city` | text | City, set by the dreamer or derived from device |
| `source_type` | text | How the dream entered the system: user_entry, bulk_import, dreambank |
| `created_at` | timestamptz | When the record was created |

## Tier 2: Derived

Computed from Tier 1 fields without subjective judgment. Given the same dream text and the same pipeline version, these fields will come out the same way. They are the most stable layer between what the dreamer wrote and what the platform infers.

| Field | Type | Description |
|---|---|---|
| `embedding` | vector(1536) | Semantic embedding of the dream text, generated by OpenAI text-embedding-3-small |
| `word_count` | integer | Word count of dream_text |
| `source_collection` | text | Which corpus the dream belongs to, for DreamBank dreams |
| `source_gender` | text | Gender as recorded in the source corpus, for DreamBank dreams |
| `resolved_symbols` | text[] | Symbol slugs resolved from enrichment fields |
| `auto_symbols` | text[] | Symbol slugs assigned by the automated pipeline |
| `atlas_fragment` | text | The short anonymised fragment offered to the collective atlas |
| `atlas_symbol` | text | The dominant symbol slug for the atlas dot, assigned by classify-atlas-fragment |
| `recurrence_ref_id` | text | Reference to the earlier dream this one recurs from, if identified |
| `packet5_id` | uuid | The 5-dream packet this dream belongs to |
| `packet10_id` | uuid | The 10-dream packet this dream belongs to |
| `enrichment_sources` | jsonb | Per-field provenance tracking: which fields were set by the user and are locked from enrichment |
| `published_to_atlas` | boolean | Whether the dreamer has offered this dream to the collective atlas |
| `deleted_at` | timestamptz | Soft delete timestamp |

## Tier 3: Interpretive

Generated by a language model reading the dream text. These fields are offered as background context for Marie-Louise, not as conclusions about the dreamer. They inform her attention without determining it, which is the only honest way to use them.

The fields in this tier are the ones most likely to be wrong in interesting ways. A dream that resists clean classification will produce low confidence scores and ambiguous motifs, and that ambiguity is itself worth attending to. The number attached to `interpretation_confidence` is a rough signal.

| Field | Type | Description |
|---|---|---|
| `primary_archetype_hint` | text | The dominant Jungian archetype present in the dream. Examples: shadow, anima, wise old man, child, trickster, self |
| `recurring_motif_1` | text | The primary symbolic image or action in the dream. Single word or short phrase |
| `recurring_motif_2` | text | The secondary motif, if present |
| `recurring_motif_3` | text | The tertiary motif, if present |
| `dominant_emotion` | text | The dominant emotion of the dream, from a constrained vocabulary: fear, joy, neutral, confusion, sadness, love, anger, awe |
| `emotional_tone_ml` | text | A free-text characterisation of the emotional register, generated by the enrichment pipeline |
| `vividness_ml` | integer | Vividness as assessed by the model, 1 to 5 |
| `shadow_present` | boolean | Whether shadow material is detected in the dream |
| `anima_animus_present` | boolean | Whether anima or animus figures are detected |
| `self_symbol_present` | boolean | Whether a Self symbol is detected: mandala, round wholeness, the centre |
| `individuation_signal` | text | A soft characterisation of where this dream sits in the individuation arc |
| `numinosity_score` | float | A 0 to 1 signal of numinous quality. The hardest field to calibrate. Treat with particular caution |
| `visitation_detected` | boolean | Whether the dream has qualities associated with visitation dreams |
| `lucidity_level` | text | Degree of lucidity during the dream: none, partial, full |
| `atmosphere_tags` | text[] | Atmospheric qualities detected: uncanny, sacred, liminal, oppressive, expansive |
| `emotional_arc` | text | How the emotional tone moves through the dream from beginning to end |
| `image_held` | text | The single image Marie-Louise is holding from this dream, identified during conversation |
| `enrichment` | jsonb | The full enrichment output from the pipeline, including similarity matches against the DreamBank corpus |
| `interpretation_confidence` | float | The model's confidence in its Tier 3 classifications, 0 to 1 |

## Tier 4: Guardrail

The provenance layer. These fields record where the Tier 3 classifications came from, when they were generated, and which pipeline version produced them. They exist so that a field can be questioned, traced, and if necessary corrected. A schema without this layer is asking to be trusted without giving any reason for the trust.

| Field | Type | Description |
|---|---|---|
| `pipeline_version` | text | The version of the enrichment pipeline that generated the Tier 3 fields |
| `enrichment_sources` | jsonb | Per-field provenance: which fields came from the user, which from the pipeline, which are locked |
| `analysis_version` | text | The embedding model version used |
| `enriched_at` | timestamptz | When the enrichment was last run on this dream |
| `enrichment_version` | text | The schema version at the time of enrichment |

## A note on numinosity

Numinosity is the field that most resists this schema. Rudolf Otto's original concept, which Jung drew on extensively, points at something that is by definition not fully articulable: the quality of encounter with something that exceeds ordinary categories, that is simultaneously fascinating and overwhelming, that leaves a residue in the psyche that does not dissolve on waking.

Assigning a number to this is philosophically uncomfortable. The field exists because numinous dreams are genuinely different in their effects from non-numinous ones, and because having some signal of their presence, however imperfect, is more useful than having none. But it should be used with more caution than any other field in the schema. A `numinosity_score` of 0.8 does not mean the dream was numinous. It means a language model, reading the text, found qualities that are associated with numinous reports in the corpus. That is a much weaker claim.
